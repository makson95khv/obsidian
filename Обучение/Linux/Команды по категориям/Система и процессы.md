`uname` - вывод информации о системе (через опции можно вывести разную информацию)
	`-r` - выведет версию ядра
`dmesg` - вывести сообщения ядра (что-то типа лога)
`pstree` - дерево процессов
`lsb_release` - вывод информации о дистрибутиве (`-a` - ключ all)
`uptime` - покажет, сколько времени запущена система
- цифры в конце - это общая загрузка системы (последняя минута/последние 5 минут/последние 15 минут)
`hostname` - покажет имя ПК
`echo $SHELL` - покажет какой shell
`w` - выведет залогиненных пользователей
`&` - в конце команды - запустит эту команду в фоне и отпустит терминал
`jobs` - выведет процессы, запущенные в фоне
`fg` - переключится на процесс (номер процесса из jobs)
`bg` - запустить в фоне
`nohup` - процессы будут запущены даже при выходе из системы
`ulimit -u` - ограничение процессов на пользователя
`ps` - выведет список процессов системы (`ps -ef` - полный список)
- `-aux` - выведет все процессы запущенные от пользователя
- `-ps -eo user,pid,pcpu,nice,comm` - выведет указанные столбцы
`kill [id]` - завершить процесс
- `killall [name]` - завершит все процессы с таким именем
`pgrep [name] -l` - выведет все процессы с именем и id
- `-u` - процессы пользователя
`pkill` - так же может завершить процессы по определенным параметрам
`top` - диспетчер задач для Линукс
	Некоторые команды и сочетания в `top`
		`shift + p` - сортировка по ЦПУ
		`shift + m` - сортировка по оперативной памяти
		`h` - выведет подсказку по горячим клавишам
`journalctl -e` - выведет логи от разных юнитов
`free` - выводит используемую и свободную оперативную память
`pstree` - выведет дерево всех запущенных процессов
`screen` - что-то типа оконного менеджера
- `-ls` - выведет список скринов
- `-r` - перейти на скрин
`nice -n -19 [name]` - запустит процесс с установленным уровнем nice
`renice -10 -p [id]` - изменит приоритет процессу на -10 
## LS (старая утилита)
- `lscpu` - информация о процессоре
- `lspci` - информация об устройствах, подключенных к pci
- `lsusb` - информация об устройствах, подключенных к usb
- `lshw` - детальная информация о комплектующих
- `lsmod` - информация о модулях ядра 
## UDEV (можно выводить инфо)
- `info` - получение информации из БД udev
- `trigger` - запросить события для устройства
- `settle` - дождаться завершение обработки
- `control` - управление демоном
- `monitor` - следить за событиями
- `test` - симулировать запуск события

`nice` - для *выставления* приоритета процессу (от -20 до 19, чем число выше, тем больше программа будет уступать другим. `renice` - для *изменения* приоритета)
